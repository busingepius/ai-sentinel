# ai-sentinel

## AI-Assisted Zero-Trust API Defense for Spring Boot

---

# 1. Executive Summary

**ai-sentinel** is an open-source Spring Boot security framework that introduces lightweight, AI-assisted anomaly detection and adaptive zero-trust enforcement directly into API request processing.

Traditional API protection mechanisms rely primarily on:

* Static rate limits
* Rule-based WAF policies
* Binary authentication decisions

These approaches struggle to detect evolving patterns such as:

* Credential stuffing
* Automated scraping
* Parameter tampering
* Low-and-slow reconnaissance
* Session replay anomalies

ai-sentinel addresses this gap by:

1. Learning behavioral baselines for API clients and endpoints
2. Computing contextual threat scores per request
3. Dynamically adjusting trust and enforcement policies

The framework operationalizes principles outlined in:

* **NIST SP 800-207 – Zero Trust Architecture**
* **CISA Zero Trust Maturity Model**

Its goal is to make AI-assisted API protection accessible to mainstream Spring Boot systems in finance, retail, healthcare, and other security-sensitive environments.

---

# 2. Design Philosophy

## 2.1 Library-First Architecture

ai-sentinel is implemented as a **Spring Boot starter**, not a standalone microservice.

### Why?

* Enables 10-minute adoption via Maven dependency
* Avoids network latency and operational overhead
* Encourages real-world usage
* Simplifies evaluation
* Maximizes adoption probability

A centralized microservice deployment model may be introduced in future versions, but v1 prioritizes adoption and simplicity.

---

# 3. Problem Statement

Modern APIs operate in hostile environments.

While authentication verifies identity, it does not continuously evaluate behavioral trust.

Common gaps include:

* Valid JWT tokens used in anomalous patterns
* Automated abuse from rotating IP pools
* Subtle deviation from normal endpoint usage
* Attackers staying below static rate thresholds

Most systems treat trust as static.

Zero Trust principles require:

> Continuous evaluation of contextual risk.

ai-sentinel provides that capability within the application layer.

# 4. Core Capabilities

## 4.1 Behavioral Feature Extraction

For each incoming request, the framework extracts structured features, including:

* Requests per identity per time window
* Endpoint entropy and traversal patterns
* Token age and reuse characteristics
* Parameter count deviation
* Payload size anomalies
* Header fingerprint consistency
* IP bucket behavior

All features are privacy-aware and avoid raw PII storage.

---

## 4.2 Lightweight Machine Learning Engine (V1)

V1 includes production-credible ML techniques:

### Statistical Baseline Modeling

Maintains rolling mean and standard deviation per identity and endpoint.

Anomaly score calculated via:

[
z = \frac{x - \mu}{\sigma}
]

### Isolation Forest (Unsupervised Anomaly Detection)

* Trained on normal behavioral samples
* CPU-efficient
* No labeled attack data required
* Suitable for real-time scoring

This allows ai-sentinel to detect deviations without predefined attack signatures.

---

## 4.3 Contextual Threat Scoring

Each request receives a composite threat score derived from:

* Behavioral anomaly score
* Token risk indicators
* Endpoint sensitivity weighting
* Burst detection metrics

The system produces a continuous risk spectrum rather than binary decisions.

---

## 4.4 Adaptive Enforcement Policies

Based on score thresholds:

| Risk Level | Enforcement Action       |
| ---------- | ------------------------ |
| Low        | Allow                    |
| Moderate   | Allow + enhanced logging |
| Elevated   | Adaptive throttling      |
| High       | Temporary block          |
| Critical   | Quarantine session       |

This implements dynamic zero-trust evaluation consistent with NIST guidance.

---

## 4.5 Adaptive Rate Limiting

Unlike static rate limiting:

* Thresholds adjust based on anomaly score
* Suspicious sessions receive tighter limits
* Normal sessions avoid unnecessary throttling

This improves detection precision and reduces false positives.

---

## 4.6 Security Telemetry & Observability

ai-sentinel emits structured events:

* ThreatScored
* AnomalyDetected
* PolicyActionApplied
* QuarantineStarted

Telemetry integrates with:

* JSON structured logs
* Prometheus metrics
* OpenTelemetry
* SIEM systems

This ensures operational transparency and auditability.

---

## 4.7 Operational Dashboard

The optional dashboard visualizes:

* Threat score distribution over time
* Top anomalous endpoints
* Enforcement actions
* Latency overhead

This supports both operational teams and evaluation studies.
